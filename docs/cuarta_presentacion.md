# Objetivos del proyecto
El objetivo general del proyecto se establece como:

- Insertar un componente de depurabilidad a las actividades de la arquitectura M32 en la clase de arquitectura de computadores

Para lograr este objetivo, se establecen los siguientes objetivos generales:

- Implementar el procesador M32 en Logisim Evolution
- Agregar documentación sobre cada componente a la implementación de M32 en Logisim Evolution
- Facilitar la visualización de estados dentro del procesador M32 en Logisim Evolution

# Primeros requerimientos de software
Requisitos funcionales:

- Implementación de M32 en Logisim Evolution
- Documentación de componentes principales de M32
- Documentación del automata de la Unidad de Control de M32

Requisitos no functionales:

- Limpieza en el cableado entre componentes
- Documentación clara
- Separación modular entre componentes

# Planificación de requerimientos
Para el desarrollo del proyecto se propone la siguiente planificación:
- Semana 1 a 4: Implementación de M32 en Logisim Evolution
- Semana 5: Primeras pruebas de usabilidad
- Semana 6: Correcciones a la version inicial
- Semana 7: Documentación de componentes
- Semana 8: Segunda prueba de usabilidad
- Semana 9: Correcciones a la segunda versión
- Semana 10: Documentación del automata de la unidad de control
- Semana 11: Última prueba de usabilidad
- Semana 12: Correcciones finales
- Semana 13 a 16: Escritura de programas de ejemplo para M32
- Semana 17 a 20: Documentación del proyecto (informe final)

# Conclusiones
En conclusión este proyecto contempla:
- Implementar M32 y su respectiva documentación dentro de Logisim Evolution
- Probar (en al menos tres instancias) el uso de esta herramienta en clases, o con docentes de la asignatura
- Documentar el proceso de desarrollo del proyecto
